## break_reminder.py 优化
对 break_reminder.py 进行了8项优化：
1. 子线程弹窗改用 pyqtSignal → 主线程槽函数弹出（修复 Qt 崩溃风险）
2. 锁屏逻辑：改为暂停计时 + 解锁后补偿 deadline，不再直接 quit（修复功能错误）
3. 计时改用 time.time() 基准 deadline，去除 sleep 循环累计误差
4. 补全声音/通知调用（show_notification 之前从未被调用）
5. is_screen_locked() 拆分为3个独立函数，加方法编号缓存（减少 subprocess 开销）
6. 修复 ReminderDialog 倒计时初始少显示1秒（改为先展示再递减）
7. 提取 center_widget() 工具函数，消除三处重复定义

## break_reminder.py 修复锁屏检测根本故障（第十轮）
- **根本原因**：`_check_cgsession()` 中 `/usr/bin/python3`（macOS 自带 Python 3.9.6）没有安装 `pyobjc` 的 Quartz 模块，import 失败后返回 `None`，fallback 到 `pmset`/`ioreg`，而这些方法检测不到用户锁屏（只检测显示器关闭）
- **字典 key 错误**：`CGSessionCopyCurrentDictionary()` 返回的 key 是 `kCGSSessionOnConsoleKey`，代码中错误使用 `OnConsole`，导致始终取到 `False`，`not False = True`，程序误判为始终锁屏
- **修复方案**：
  1. Python 解释器路径从硬编码 `/usr/bin/python3` 改为自动探测：依次尝试 `/usr/local/bin/python3`、`/usr/bin/python3`、`sys.executable`
  2. 字典 key 从 `"OnConsole"` 改为 `"kCGSSessionOnConsoleKey"`
  3. 同步修复 `_check_quartz_display_sleep()` 的 Python 路径硬编码问题
- **测试验证**：用 Python 3.13.12 (managed) 运行时，subprocess 正确调用 `/usr/local/bin/python3` (Python 3.14.3, 有 Quartz)，`is_screen_locked()` 返回 False（未锁定），结果正确
- **旧进程清理**：杀掉了 PID 4954 的旧进程（旧代码），清理了 PID 文件

## break_reminder.py 彻底替换锁屏检测方法（第十一轮）
- **用户验证失败**：锁屏 34 秒后解锁，倒计时从 01:09 继续到 01:43，说明锁屏检测仍失效
- **根本原因**：`kCGSSessionOnConsoleKey` 在锁屏时仍然是 `True`，CGSessionCopyCurrentDictionary 检测的是"用户是否在 GUI 会话中"，不是"屏幕是否锁定"。之前整个 `_check_cgsession()` 概念错误
- **彻底替换检测方法**：
  - 删除 `_check_cgsession()` 和 `_check_quartz_display_sleep()`（依赖外部 Python + 概念错误）
  - 新增 `_check_applescript_screensaver()`：用 AppleScript `tell System Events to return exists process "ScreenSaverEngine"` 直接查询系统事件
  - 新增 `_check_applescript_loginwindow()`：用 AppleScript 检查 `loginwindow` 是否为前台进程
  - 保留 `_check_screensaver_engine()`（pgrep 备选）和 `_check_pmset()`、`_check_ioreg_lock()`
  - **移除方法缓存** `_screen_check_method`：锁屏/解锁时方法可靠性可能变化，缓存可能导致错过检测，每次调用都顺序尝试所有方法
- **检测优先级**：AppleScript ScreenSaverEngine → AppleScript loginwindow 前台 → pgrep ScreenSaverEngine → pmset → ioreg

## break_reminder.py 用 log stream 实现锁屏检测（第十二轮）
- **所有之前的锁屏检测方案全部失败**（Quartz CGSession、AppleScript、pgrep、pmset、ioreg、NSDistributedNotificationCenter、NSWorkspace通知、CGDisplayRegisterReconfigurationCallback）
- **成功方案**：通过 `/usr/bin/log stream --predicate 'process == "loginwindow"' --info` 实时监控 loginwindow 日志
  - 锁屏关键字：`going inactive, create activity semaphore`
  - 解锁关键字：`closing and releasing _screenLockWindowController`（最可靠）
- **新增 `ScreenLockMonitor` 类**：后台线程运行 log stream 子进程，逐行解析输出，检测到锁屏/解锁事件时调用回调
  - `on_lock` 回调：标记 `_screen_locked = True`，暂停计时
  - `on_unlock` 回调：标记 `_screen_locked = False`，重置计时器从0开始
  - log stream 进程异常退出后自动5秒重启
- **移除旧的 `check_and_reset_if_abnormal()`**：不再需要异常时间检测（`interval * 2` 方案）
- **移除 `DEBUG_ABNORMAL` / `debug_abnormal()`**：替换为 `DEBUG_LOCK` / `debug_lock()`
- **新增信号**：`reset_timer_signal`（解锁重置）、`pause_timer_signal`（锁屏暂停）
- **reminder_loop 改造**：`_screen_locked` 为 True 时跳过倒计时检查，只 sleep 等待
- **注意**：log stream 在沙箱环境中无法运行（Cannot run while sandboxed），但作为 LaunchAgent 或终端直接运行时正常
